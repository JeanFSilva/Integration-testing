	Quando testamos conexão com Banco de Dados, uma boa prática é fazer isso dentro de um contexto de transação.
		@Before
		public void antes(){
			//Geralmente colocamos aqui
			//Criação da sessão ou conexão com o Banco de dados.
			//Instanciação dos Daos
			//E beginTransaction
			
			session = new CriadorDeSessao().getSession();// Cria conexão com o banco de dados de verdade
			usuarioDao = new UsuarioDao(session);
			
			session.beginTransaction();
		}
		
		@After
		public void depois() {
			//Geralmente colocamos aqui
			//RollBack da transação
			//Fechamos a conexão com o banco de dados.
			session.getTransaction().rollback(); //quando acabar o teste, o banco vai voltar atrás tudo o que aconteceu. 
												//O banco de dados vai estar sempre limpo. Um teste não vai atrapalhar o outro
			session.close();
		}
		Os Testes tem que ser independentes.
		Um teste não pode sujar o cenário do outro.
		Abrir e fechar a transação é uma ótima maneira pra resolver esse problema.
		
		Teste de integração são mais caros porque:
			Bate no banco de dados.
			Se comunica com outro sistema.
			
		Mante o banco de dados limpo é fundamental para que a bateria testes cresça com qualidade.
